Overview

RelativisticOrbit integrates the trajectory of a test particle — a planet or spacecraft of negligible mass — moving in the gravitational field of a central body in the context of Einstein's general theory of relativity. The central body is modelled as a non-rotating, spherically symmetric mass (a Schwarzschild black hole or compact star), and the orbit is computed in the equatorial plane of the Schwarzschild geometry.

Unlike the purely Newtonian Binary and MercPert programs, RelativisticOrbit incorporates the leading post-Newtonian correction to the radial force — a term proportional to \(1/r^4\) — which causes an otherwise elliptical orbit to precess. This precession is the famous general-relativistic prediction confirmed for Mercury in 1915, one of the first observational tests of general relativity.

The program uses the same predictor–corrector integration scheme as Binary and MercPert. The user specifies the initial position and velocity of the test particle, the number of orbits to simulate, and the accuracy tolerances, and the program produces an orbital plot in the \(xy\)-plane with the Schwarzschild horizon drawn as a reference circle.

Scope: The GR correction used here is the effective post-Newtonian acceleration derived from the Schwarzschild metric. It is exact for equatorial geodesics in Schwarzschild geometry — not a perturbative approximation — but it treats the central body as a point mass and neglects radiation reaction, frame-dragging (Kerr rotation), and any tidal or finite-size effects.

Quick Start

Requirements

Python 3.9 or later with matplotlib:

pip install matplotlib

Running the Program

from driver_relativistic_orbit import run_relativistic_orbit, RelativisticOrbitParams
from plot_relativistic_orbit import plot_relativistic_orbit

params = RelativisticOrbitParams(
    x_init     = 1.0e11,    # initial x-position (m);  start on the x-axis
    u_init     = 8862.0,    # initial y-velocity  (m/s); > 0 → counter-clockwise
    dt         = 500.0,     # initial time-step   (s)
    max_steps  = 2_000_000, # safety cap on total integration steps
    max_orbits = 5,         # stop after this many complete orbits
    eps1       = 1.0e-3,    # fractional acceleration change allowed per step
    eps2       = 1.0e-4,    # corrector-loop convergence tolerance
)

result = run_relativistic_orbit(params)
if result.fell_into_hole:
    print(f"Particle crossed the Schwarzschild horizon after "
          f"{result.final_step} steps.")
else:
    print(f"Integration complete: {result.n_orbits:.0f} orbit(s) "
          f"in {result.final_step} steps.")

plot_relativistic_orbit(result)
cd GFTGU-Programs/RelativisticOrbit
python main.py

The default configuration places the test particle at \(4 \times 10^{10}\,\mathrm{m}\) from the central mass (\(M = M_\odot\)) with a transverse velocity of \(3 \times 10^4\,\mathrm{m\,s^{-1}}\) and integrates for 5 complete orbits. The Schwarzschild radius for one solar mass is \(r_g = 2GM_\odot/c^2 \approx 2{,}953\,\mathrm{m}\), far inside the orbit, so the particle is safely in the weak-field regime where precession is small but measurable.

Adjusting the Simulation

All parameters are set in RelativisticOrbitParams in main.py:

  • x_init — initial x-position (m); particle starts on the x-axis
  • u_init — initial y-velocity (m s−1); positive gives counter-clockwise orbit
  • dt — initial timestep (s); halved automatically when accuracy demands it
  • max_steps — safety cap on total integration steps
  • max_orbits — stop after this many complete orbits
  • eps1 — vector-norm fractional acceleration change allowed per step
  • eps2 — corrector-loop convergence tolerance on position change

No other file needs editing for typical use.

Program Files

main.py
Entry Point & Configuration

Defines the RelativisticOrbitParams instance with all initial conditions and run parameters, calls integrate_relativistic_orbit(params) to integrate the trajectory, and passes the result to plot_relativistic_orbit(result) for display. Prints a one-line summary reporting either the number of completed orbits or a horizon-crossing notification. All user-adjustable settings live here; no other file needs editing for typical use.

physics_relativistic_orbit.py
Physics Engine

Defines the physical constants \(GM_\odot = 1.327\times10^{20}\,\mathrm{m^3\,s^{-2}}\), \(c^2 = 8.98755\times10^{16}\,\mathrm{m^2\,s^{-2}}\), and \(r_g = 2GM_\odot/c^2\) (the Schwarzschild radius for one solar mass). Provides two functions: kepler_constants(x_init, u_init) computes the orbital constants \(K\) and \(Q\) from the initial conditions; schwarzschild_acceleration(x, y, K, Q) evaluates the general-relativistic effective acceleration at the current position.

driver_relativistic_orbit.py
Integration Driver

Defines the RelativisticOrbitParams and RelativisticOrbitResult dataclasses and the function run_relativistic_orbit(params). The driver integrates the orbit using the same predictor–corrector scheme as Binary and MercPert, with adaptive timestep halving on the acceleration-change test (\(\varepsilon_1\)) and corrector convergence on the position-change test (\(\varepsilon_2\)). It counts completed orbits by tracking sign changes in \(y\), stops when max_orbits is reached or the particle crosses inside the Schwarzschild horizon, and returns the full trajectory plus a circle of horizon points for the plot.

plot_relativistic_orbit.py
Visualisation

Provides plot_relativistic_orbit(result), which renders a matplotlib figure showing the orbital trajectory in blue and the Schwarzschild horizon as a red filled circle at the origin. Because the true horizon radius (~2,953 m) is many orders of magnitude smaller than a typical orbital radius, the circle is drawn with radius = max(HORIZON_RADIUS, 2_pixels_in_data_units) so it is always visible regardless of zoom level. The legend is placed below the axes area so it never obscures the trajectory. Equal aspect ratio is enforced.

Schwarzschild Geometry

The Schwarzschild metric describes the curved spacetime outside any spherically symmetric, non-rotating mass \(M\). In Schwarzschild coordinates \((t, r, \theta, \phi)\) with \(\theta = \pi/2\) (equatorial plane), the metric reads:

(1) \[ ds^2 \;=\; -\!\left(1 - \frac{r_g}{r}\right)c^2\,dt^2 + \left(1 - \frac{r_g}{r}\right)^{-1}\!dr^2 + r^2\,d\phi^2 \]

where \(r_g = 2GM/c^2\) is the Schwarzschild (gravitational) radius. For one solar mass:

(2) \[ r_g \;=\; \frac{2\,GM_\odot}{c^2} \;=\; \frac{2 \times 1.327\times10^{20}}{8.98755\times10^{16}} \;\approx\; 2{,}953\;\mathrm{m} \]

The surface \(r = r_g\) is the event horizon: the boundary beyond which no signal can escape to infinity. A test particle that crosses inside \(r = r_g\) cannot return to the outer region.

Coordinates vs. geometry: Schwarzschild coordinates are singular at \(r = r_g\), but the physics is not — the event horizon is a coordinate singularity, not a physical one. The program detects horizon crossing by checking whether the particle's coordinate radius \(r = \sqrt{x^2 + y^2}\) falls below HORIZON_RADIUS and terminates the integration at that point.

General-Relativistic Effective Acceleration

For a test particle moving in the equatorial plane of a Schwarzschild black hole, the radial equation of motion derived from the geodesic equations can be recast as an effective Newtonian-like force with a correction term. In Cartesian coordinates \((x, y)\) with the central mass at the origin, the GR effective acceleration is:

(3) \[ \ddot{\mathbf{r}} \;=\; -\frac{GM}{r^3}\left(1 + \frac{Q}{r^2}\right)\mathbf{r} \]

where \(r = \sqrt{x^2 + y^2}\) and \(Q\) is a constant determined by the orbit's angular momentum (see below). In component form:

(4) \[ \ddot x \;=\; -\frac{GM}{r^3}\!\left(1 + \frac{Q}{r^2}\right)x, \qquad \ddot y \;=\; -\frac{GM}{r^3}\!\left(1 + \frac{Q}{r^2}\right)y \]

Comparing with the Newtonian acceleration \(\ddot{\mathbf{r}} = -GM\mathbf{r}/r^3\), the GR correction is the additional factor \(Q/r^2\). This correction is negligible when \(r \gg \sqrt{Q}\) (the weak-field regime) but grows rapidly as the particle approaches the central mass. At large \(r\) the orbit approximates a Keplerian ellipse; close in, the correction causes the orbit to precess.

This effective acceleration is implemented in schwarzschild_acceleration(x, y, K, Q) in physics_relativistic_orbit.py as: 

r2     = x**2 + y**2
r3     = r2 ** 1.5
factor = -GMSOLAR / r3 * (1.0 + Q / r2)
ax     = factor * x
ay     = factor * y

The Orbital Constants K and Q

The constant \(Q\) in the GR acceleration (eq. 3) is determined by the specific angular momentum of the orbit. For a test particle starting on the \(x\)-axis (\(y = 0\)) with initial transverse velocity \(\dot y = v_y\) and initial radial position \(x = x_0\), the specific angular momentum per unit mass is \(L = x_0\,v_y\). The program uses a scaled version:

(5) \[ K \;=\; \frac{1}{2}\,x_0\,v_y \]

where the factor of \(\tfrac{1}{2}\) follows Schutz's convention (half the specific angular momentum). The GR correction parameter is then:

(6) \[ Q \;=\; \frac{12\,K^2}{c^2} \]

These are computed once at the start of the integration by kepler_constants(x_init, u_init) and held fixed for the entire run. The factor of 12 in equation (6) comes from the Schwarzschild geodesic equation in the weak-field limit and encodes the leading-order GR correction to Keplerian motion.

Physical interpretation of Q: The characteristic length scale \(\sqrt{Q} = \sqrt{12}\,K/c\) is the radius at which the GR correction term \(Q/r^2\) becomes comparable to unity. Inside this radius the orbit deviates strongly from a Keplerian ellipse. For a Mercury-like orbit, \(\sqrt{Q} \approx 3{,}500\,\mathrm{m}\) — much smaller than Mercury's orbital radius of \(5.8\times10^{10}\,\mathrm{m}\) — confirming that GR effects on Mercury are a small but measurable perturbation.

Orbital Precession

In Newtonian gravity the orbit of a test particle in a central \(1/r^2\) force field is a closed ellipse that repeats exactly after each period. The GR correction term \(Q/r^4\) in the radial force breaks this closure: the ellipse rotates in its own plane by a small angle \(\Delta\phi\) per orbit, known as the perihelion precession.

For a Schwarzschild geometry the exact formula for the precession per orbit in the weak-field limit is:

(7) \[ \Delta\phi \;=\; \frac{6\pi\,GM}{a(1-e^2)\,c^2} \]

where \(a\) is the semi-major axis and \(e\) is the eccentricity of the unperturbed Keplerian ellipse. For Mercury, \(\Delta\phi \approx 43^{\prime\prime}\) per century — one of the first precision confirmations of general relativity.

RelativisticOrbit makes this precession directly visible: after even a few orbits the aphelion point of the trajectory shifts measurably in the orbital plot, rotating counter-clockwise with each pass.

Measuring precession numerically: Run the simulation for many orbits (increase max_orbits) and compare the angular position of successive perihelion passages (closest approach points in the plot). The difference in angle per orbit gives the numerical precession rate, which can be compared with equation (7) evaluated for the chosen initial conditions.

Event Horizon and Plunge Orbits

If the initial velocity is too low for a given orbital radius, or if the orbit decays close to the central mass, the test particle crosses the innermost stable circular orbit (ISCO) and plunges inside the Schwarzschild horizon. In Schwarzschild geometry the ISCO radius is:

(8) \[ r_{\mathrm{ISCO}} \;=\; 3\,r_g \;=\; \frac{6\,GM}{c^2} \]

Inside the ISCO there are no stable circular orbits; any particle at \(r < r_{\mathrm{ISCO}}\) that is not moving outward fast enough will spiral inward.

The driver monitors \(r = \sqrt{x^2 + y^2}\) at every step and terminates the integration immediately if \(r < \texttt{HORIZON\_RADIUS} = r_g\). The RelativisticOrbitResult includes a flag indicating whether the run ended by horizon crossing or by reaching max_orbits, and the plot always shows the horizon as a red circle at the origin so the user can see how close the orbit came to the boundary.

Exploring plunge orbits: Reduce u_init until the orbit just grazes the ISCO — the trajectory will spiral inward over several orbits and eventually terminate at the horizon. The critical velocity for a circular orbit at radius \(r_0\) is \(v_c = \sqrt{GM/r_0}\); setting u_init significantly below \(v_c\) will produce a plunge orbit within a few revolutions.

Predictor–Corrector

RelativisticOrbit uses the same two-stage predictor–corrector as Binary and MercPert. Let superscript \(n\) denote values at time \(t\) and \(n+1\) values at \(t + \Delta t\). The current GR acceleration is \(\mathbf{a}^n = \texttt{schwarzschild\_acceleration}(x^n, y^n, K, Q)\).

Stage 1 — Euler Predictor

First estimates of the state at \(t + \Delta t\) are obtained by a simple Euler step:

(9) \[ \tilde{\mathbf{v}} \;=\; \mathbf{v}^n + \mathbf{a}^n\,\Delta t, \qquad \tilde{\mathbf{r}} \;=\; \mathbf{r}^n + \mathbf{v}^n\,\Delta t \]

The predicted acceleration is then evaluated: \(\tilde{\mathbf{a}} = \texttt{schwarzschild\_acceleration}(\tilde x, \tilde y, K, Q)\).

Stage 2 — Averaged-Acceleration Corrector

The corrected velocity and position average the start-of-step and predicted accelerations or velocities:

(10) \[ \mathbf{v}^{n+1} \;=\; \mathbf{v}^n + \tfrac{1}{2}\!\left(\mathbf{a}^n + \tilde{\mathbf{a}}\right)\Delta t \]
(11) \[ \mathbf{r}^{n+1} \;=\; \mathbf{r}^n + \tfrac{1}{2}\!\left(\mathbf{v}^n + \mathbf{v}^{n+1}\right)\Delta t \]

The corrector is applied once per accepted step. The GR acceleration is an autonomous function of position only (no explicit time dependence), so the predictor and corrector evaluations differ only in their spatial argument.

Adaptive Timestep

Two sequential checks govern the timestep. Both test the maximum fractional change in the relevant quantity across the step.

Acceleration Check (eps1)

The fractional change in acceleration magnitude between the predictor and the corrector is tested component-wise:

(12) \[ \delta_a \;=\; \max\!\left( \frac{|\tilde a_x - a_x^{n+1}|}{|a_x^{n+1}|},\; \frac{|\tilde a_y - a_y^{n+1}|}{|a_y^{n+1}|} \right) \]

If \(\delta_a > \varepsilon_1\) the step is rejected, the timestep is halved, and the predictor–corrector is retried:

(13) \[ \Delta t \;\leftarrow\; \tfrac{1}{2}\,\Delta t \qquad (\delta_a > \varepsilon_1) \]

Position Corrector Check (eps2)

If the acceleration check passes, the fractional change in position between the predictor and the corrector is compared with \(\varepsilon_2\):

(14) \[ \delta_r \;=\; \max\!\left( \frac{|\tilde x - x^{n+1}|}{|x^{n+1}|},\; \frac{|\tilde y - y^{n+1}|}{|y^{n+1}|} \right) \]

If \(\delta_r > \varepsilon_2\) the timestep is again halved and the step retried. If both checks pass the step is accepted and \(t\) is advanced by \(\Delta t\).

Timestep collapse near the horizon: As the test particle approaches the Schwarzschild horizon the acceleration grows very steeply (the GR correction \(Q/r^2\) diverges as \(r \to 0\)). This forces repeated halving of the timestep, which can stall the integration before the horizon is actually reached. If this occurs, loosen eps1 or reduce x_init to start further from the horizon. The RK45-based version of this program (when available) will handle this automatically.

Orbit Counting

The driver counts completed orbits by monitoring the sign of the \(y\)-coordinate. Each time the particle crosses the positive \(x\)-axis (i.e., \(y\) changes sign from negative to positive while \(x > 0\)) one complete orbit has been executed and the orbit counter is incremented. Integration terminates when the counter reaches max_orbits.

This simple criterion assumes the orbit is prograde and does not cross the origin. For highly eccentric or near-plunge orbits the sign-change detector may fire more or fewer times than expected — in those cases, monitor the plotted trajectory directly rather than relying on the orbit count.

Precessing orbits and orbit counting: Because the orbit precesses, the particle does not return to exactly the same \(\phi = 0\) line each time. The sign-change criterion still works because the orbit remains topologically closed on each pass — the particle crosses \(y = 0\) with \(x > 0\) once per revolution regardless of how much the perihelion has rotated.

Convergence Criteria

Parameter Symbol Suggested default Role
eps1 (vector-norm) \(\varepsilon_1\) \(10^{-3}\) Timestep gate. Steps where the fractional change in acceleration between predictor and corrector exceeds this value are rejected and retried at half the timestep.
eps2 \(\varepsilon_2\) \(10^{-4}\) Corrector position test. Steps where the fractional positional difference between predictor and corrector exceeds this value are rejected. Should be strictly less than \(\varepsilon_1\).
Accuracy vs. speed: Tightening both tolerances by a factor of 10 significantly increases the number of accepted steps but produces a more accurate precession rate. For a quick qualitative view of the precessing orbit the defaults are adequate; for measuring the precession angle to compare with equation (7), use \(\varepsilon_1 = 10^{-4}\) and \(\varepsilon_2 = 10^{-5}\).

Suggested Scenarios

Mercury Analogue
Set x_init = 5.8e10 m (Mercury's semi-major axis) and u_init = 4.7e4 m s−1 (Mercury's mean speed). Run for 20+ orbits. Measure the perihelion advance per orbit and compare with the predicted 43″/century.
Circular Orbit
Set u_init = sqrt(GMSOLAR / x_init) for the Keplerian circular speed at x_init. The GR correction slightly modifies the exact circular speed, so a small secular drift will still appear over many orbits.
Highly Eccentric Orbit
Reduce u_init to 30–50% of the circular speed. The large eccentricity amplifies the GR correction near perihelion, making precession visible after just 2–3 orbits.
Approaching the ISCO
Set x_init to \(6\,r_g \approx 17{,}700\,\mathrm{m}\) (the ISCO radius) and explore velocities near the circular speed. The orbit becomes unstable and spirals inward, terminating at the horizon.
Plunge Orbit
Use x_init = 4.0e10 but set u_init to 30% of the circular speed. The highly eccentric orbit will plunge through the ISCO on one of its first passes and terminate at the horizon — demonstrating horizon detection in the driver.
GR vs. Newtonian Comparison
Set Q = 0.0 manually in schwarzschild_acceleration() (effectively setting \(c \to \infty\)) and compare the closed Keplerian ellipse with the precessing GR trajectory for the same initial conditions.

Feature Newtonian (Binary) GR (RelativisticOrbit)
Force law \(-GM/r^2\) \(-GM/r^2 \cdot (1 + Q/r^2)\)
Orbit shape Closed ellipse Precessing ellipse
Innermost stable orbit None (any \(r > 0\) stable) \(r_\mathrm{ISCO} = 3\,r_g\)
Event horizon Not present \(r_g = 2GM/c^2\); orbit terminates if \(r < r_g\)
