Description of EarthOrbit

The EarthOrbit program computes the trajectory of a projectile launched near the Earth's surface, allowing the user to explore whether the projectile falls back to Earth or achieves orbit. The Python version faithfully follows the logic of Bernard Schutz’s original Java program from Gravity From the Ground Up, but is organized into modern, modular Python files.

Unlike CannonTrajectory, the acceleration of gravity in EarthOrbit always points toward the center of the Earth. The magnitude of the acceleration is taken to be 9.8 m/s², but its direction depends on the projectile’s position. This reflects the mathematics developed in Investigations 4.1 and 4.2 of the book.

Using EarthOrbit

The Python version of EarthOrbit is located at:
https://github.com/tomyee3/GFTGU-Programs/tree/main/EarthOrbit

The program is divided into four modules:

  1. Physics module (physics.py):
    Encodes the physical laws governing orbital motion. The only literal numbers in this module represent physical constants, such as the acceleration due to gravity and the radius of the Earth. It provides functions to compute gravitational acceleration and update rules for position and velocity.
  2. Driver module (driver.py):
    Performs the numerical integration. It updates velocity and position using the average velocity over each time-step, mirroring the constant-acceleration method used in Schutz’s Java program.
  3. Plot module (plot_orbit.py):
    Uses Matplotlib to display the trajectory and the Earth’s surface. The plot enforces equal horizontal and vertical scales so the orbit appears geometrically correct.
  4. Main module (main.py):
    The user-facing entry point. It sets example values for initial height, initial horizontal and vertical speeds, time-step, and maximum number of steps. Users may overwrite these values to experiment with different launch conditions.

To run the program: 

python main.py

The resulting plot shows both the Earth’s surface and the projectile’s trajectory. If the projectile achieves orbit, the trajectory will wrap around the Earth; otherwise it will fall back to the surface.

Understanding EarthOrbit

The Python program follows the same conceptual structure as the original Java version:

  1. Gravity depends on position.
    The acceleration always points toward the center of the Earth. If r is the radial distance from the center, then the acceleration components are: 
    ax = -g * x / r
ay = -g * y / r
  2. The loop continues while the projectile is above the surface and the step count is below the maximum.
    (r >= rEarth) and (j < maxSteps)
    This prevents infinite looping if the projectile escapes.
  3. Velocity is updated using acceleration.
    The driver keeps track of previous and current velocity components separately, just as in CannonTrajectory.
  4. Position is updated using the average velocity.
    This mirrors Schutz’s improved Euler method.
  5. Output includes both the orbit and the Earth’s surface.
    The plot module draws a circle representing the Earth and overlays the trajectory.

Suggested modifications

As in CannonTrajectory, you may refine the termination condition. Instead of stopping when the projectile is already inside the Earth, you could interpolate between the last two points to estimate the exact impact location.

You may also experiment with different time-steps or initial conditions to explore the sensitivity of orbital trajectories.
