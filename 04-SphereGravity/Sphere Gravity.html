Description of SphereGravity

The SphereGravity program computes the gravitational acceleration produced by a thin spherical shell of matter. It demonstrates Newton’s remarkable theorem:

  • Outside a spherical shell, gravity is identical to that of a point mass at the center.
  • Inside a spherical shell, the gravitational acceleration is exactly zero.

The Python version follows the logic of Bernard Schutz’s original Java program from Gravity From the Ground Up, but is organized into modern, modular Python files. The shell is divided into many small tiles, each treated as a point mass. The gravitational acceleration at various radii is computed by summing the contributions from all tiles.

Increasing the number of divisions increases accuracy. Users may also choose to output the relative difference between the numerical result and Newton’s theoretical prediction.

Using SphereGravity

The Python version of SphereGravity is located at:
https://github.com/tomyee3/GFTGU-Programs/tree/main/SphereGravity

The program is divided into three modules:

  1. Physics module (physics.py):
    Computes the gravitational acceleration produced by a spherical shell by summing contributions from many small tiles. The shell thickness and tile geometry are configurable.
  2. Driver module (driver.py):
    Performs the numerical summation over tiles and computes acceleration at many radii inside and outside the shell. It supports two output modes:
    • acceleration — the raw gravitational acceleration
    • relative difference — deviation from Newton’s prediction
  3. Plot module (plot_sphere.py):
    Uses Matplotlib to display acceleration vs radius. The plot clearly shows:
    • zero acceleration inside the shell
    • 1/r² falloff outside the shell
    • improving accuracy as tile count increases

To run the program: 

python main.py

The resulting plot shows the gravitational acceleration at radii from the center to far outside the shell.

Suggestions for playing with SphereGravity

The main way to experiment is to change the number of divisions used to tile the shell. Increasing this number improves accuracy.

Try increasing the number of divisions by factors of 10:

  • 100 divisions → relative error ≈ 3%
  • 1000 divisions → relative error ≈ 10⁻⁵
  • 10000 divisions → relative error ≈ 10⁻⁷

Choose the “relative difference” output mode to see how accuracy improves.

Understanding SphereGravity

The Python program follows the method described in Investigation 4.3. The shell is divided into tiles using latitude and longitude divisions. Each tile is treated as a point mass, and its gravitational contribution is added to the total.

The program:

  1. Computes the mass of the shell by summing tile masses.
  2. Loops over many radii (from 0 to ~100 shell radii).
  3. At each radius, sums contributions from all tiles.
  4. Optionally computes the relative difference from Newton’s prediction.

The result demonstrates Newton’s theorem with increasing accuracy as the number of tiles increases.

Suggested modifications

Newton’s “zero acceleration inside a shell” result also holds for elliptical shells. You may modify the program to compute gravity for an axially symmetric elliptical shell. This requires sampling points off the symmetry axis and adjusting tile geometry.

This is a challenging modification but illustrates a surprising and beautiful result.
