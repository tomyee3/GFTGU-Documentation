Description of Random

The Random program computes a three‑dimensional random walk, as described in Investigation 8.7 of Gravity From the Ground Up. Each step has the same length, but the direction of each step is chosen randomly. The resulting walk gradually moves away from the origin.

Random walks model many physical processes. In the book, this program is used to illustrate how photons diffuse outward from the Sun’s core. Mathematically, random walks demonstrate the important result that the expected displacement grows as the square root of the number of steps.

Using Random

The Python version of Random is located at:
https://github.com/tomyee3/GFTGU-Programs/tree/main/Random

The program is divided into four modules:

  1. Physics module (physics.py):
    Implements the random‑step generator. Each step has components: 
    dx = 2 * random() - 1
dy = 2 * random() - 1
dz = 2 * random() - 1
    producing a uniform distribution between −1 and +1 in each direction.
  2. Driver module (driver.py):
    Performs:
    • the random walk itself
    • repetition of the walk nTrials times
    • averaging of the net displacement
    • reduction of step count by factors of 2
    It produces arrays of:
    • number of steps
    • mean displacement
  3. Plot module (plot_random.py):
    Uses Matplotlib to display mean displacement vs number of steps. Log–log axes reveal the expected slope of 1/2.
  4. Main module (main.py):
    The user-facing entry point. It sets:
    • maxSteps — maximum number of steps in the longest walk
    • nTrials — number of repeated walks for averaging
    Users may overwrite these values to explore different random‑walk behaviors.

To run the program: 

python main.py

The resulting plot shows the mean displacement for walks of decreasing length (each half of the previous).

Understanding Random

The Python version follows Schutz’s original logic: three nested loops.

  1. Outer loop: Controls the number of steps in the walk. Each iteration halves the number of steps.
  2. Middle loop: Repeats the walk nTrials times to compute an average.
  3. Inner loop: Performs the random walk itself by generating random step components and accumulating displacement.

The net displacement is: 

sqrt(x*x + y*y + z*z)

The average step length is: 

stepSize / nSteps

The program outputs the scaled displacement: 

net_displacement / average_step_length

Plotting this quantity against the number of steps on log–log axes produces a straight line of slope 1/2, demonstrating the square‑root law.

Suggestions for playing with Random

Try:

  • Increasing nTrials for smoother averages
  • Changing maxSteps to explore longer or shorter walks
  • Plotting on linear axes to see the curvature
  • Plotting on log–log axes to reveal the slope of 1/2

Suggested modifications

You may experiment with:

  • changing the definition of “walk length”
  • removing the division by average step length
  • using Gaussian random numbers instead of uniform ones
  • changing dimensionality (2D or 1D walks)

All reasonable definitions still produce the square‑root dependence.
