Description of Atmosphere

The Atmosphere program computes the structure of a planetary atmosphere using the equation of hydrostatic equilibrium. Given a temperature profile, the program uses the ideal gas law to compute pressure and density as functions of altitude.

The Python version follows Bernard Schutz’s original Java program from Gravity From the Ground Up, but is organized into modern, modular Python files. Temperature profiles may be supplied for any planet or moon, allowing users to explore atmospheric structure across the Solar System.

The model is accurate except near the top of the atmosphere, where real physics becomes more complex than the ideal gas law can capture.

Using Atmosphere

The Python version of Atmosphere is located at:
https://github.com/tomyee3/GFTGU-Programs/tree/main/Atmosphere

The program is divided into four modules:

  1. Physics module (physics.py):
    Implements hydrostatic equilibrium: 
    dp/dh = -ρ g
ρ = p * q / T
    where q contains constants from the ideal gas law. It also provides interpolation of temperature values between measured points.
  2. Driver module (driver.py):
    Steps upward in altitude, computing pressure, density, and temperature at each level. It stops when pressure reaches zero.
  3. Plot module (plot_atmosphere.py):
    Uses Matplotlib to display:
    • pressure vs altitude
    • density vs altitude
    • temperature vs altitude
    Logarithmic scaling is available for pressure plots.
  4. Main module (main.py):
    The user-facing entry point. It sets:
    • planet name
    • surface gravity
    • mean molecular weight
    • surface pressure
    • temperature profile
    • output mode (pressure, density, temperature)
    Users may overwrite these values to explore different atmospheres.

To run the program: 

python main.py

The resulting plot shows the atmospheric structure up to the height where pressure reaches zero.

Suggestions for playing with Atmosphere

Try modeling the atmospheres of Earth, Venus, Mars, Titan, or any body for which you can find temperature data. Table 7.1 of the book provides several examples.

Do not attempt to model gas giants such as Jupiter, since their atmospheres are deep enough that gravity varies significantly with height. The Atmosphere program assumes constant gravity.

This program introduces the basic technique of hydrostatic equilibrium. Later programs (such as Star) generalize this method to stellar structure.

Understanding Atmosphere

The program steps upward in altitude using the equation of hydrostatic equilibrium: 

p[j] = p[j-1] - gAccel * rho[j-1] * dh

Density is computed using the ideal gas law: 

rho[j] = p[j] * q / Temp[j]

Temperature values are obtained by interpolation between measured points. Above the highest measurement, the program uses a simple power-law relation to keep the model finite.

The loop stops when pressure reaches zero. If the top of the atmosphere is not reached within the initial number of steps, the step size is increased and the calculation repeated.

Suggested modifications
