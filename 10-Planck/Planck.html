Description of Planck

The program suite called Planck implements, in Python, the computation of two properties of the Planck black-body radiation curve: the location of the maximum of the curve (the Wien peak), and the total area under the curve (proportional to the total radiated power). Unlike many of the other programs for Gravity from the Ground Up, Planck does not output a graph — it reports whichever of these two numbers the user chooses.

The maximum of the Planck curve tells us, when the appropriate scaling is applied, the wavelength at which peak emission occurs for a black body at a given temperature. This is the content of Wien's displacement law. The area under the curve tells us how much energy the black body radiates in total, which is governed by the Stefan–Boltzmann law. The Planck function is studied in Chapter 10 of Gravity from the Ground Up; Investigations 10.1, 10.2, and 10.3 are directly relevant to this program.

The original Java implementation (by Bernard Schutz, version 1.0, © 2003) has been faithfully ported to Python. The logic, variable names, approximation thresholds, and step-counting approach are preserved exactly, with the code reorganised into three focused modules following modern Python conventions.

Chapter Reference The physics background for this program is in Chapter 10 of Gravity from the Ground Up. Investigations 10.1, 10.2, and 10.3 guide you through using the program and interpreting its output in terms of real stellar temperatures and luminosities.

Using Planck

Run the program by executing main.py from the command line or from an IDE: 

# from the Planck/ directory:
python main.py

There are two user-adjustable parameters near the top of main.py:

ParameterDefault valueMeaning
n_steps 20 000 Number of steps across the domain of x. More steps → finer sampling → more precise answers. 20 000 is ample for six significant figures.
output_type "maximum" What the program reports. Set to "maximum" to find the Wien peak (xpeak and ypeak), or to "area" to compute the total area under the curve.

To change the output, edit main.py and replace "maximum" with "area" (or vice versa), then re-run. The program prints its result to the terminal. For example, with the default settings you should see output similar to: 

Location of maximum of Planck function:
x_peak = 4.9651...
y_peak = 21.201...

The value of xpeak ≈ 4.965 is the dimensionless location of the Wien peak and can be converted to a physical wavelength for any temperature — see Peak Location below and Investigation 10.2 in the book.

Prerequisites Python 3.9 or later is required. No third-party libraries are needed beyond the Python standard library; the code uses only built-in math functions.

Understanding Planck

The program is a direct implementation of the procedure described in Investigations 10.2 and 10.3. A loop steps through values of the dimensionless variable x from xmin = 0.01 to xmax = 100. At each step it evaluates the Planck function (using the appropriate exact formula or approximation), tests whether a new maximum has been found, and accumulates the area under the curve by the trapezoidal rule.

The Dimensionless Variable x

The Planck spectral radiance depends on temperature T and wavelength λ through the combination hc/(λkBT), or equivalently on frequency ν through hν/(kBT). Schutz's treatment introduces the dimensionless variable:

(1) x = hc / (λ kBT)  =  hν / (kBT) h = Planck constant,   c = speed of light,   kB = Boltzmann constant,   T = absolute temperature (K)

In terms of x, the spectral energy density of black-body radiation (up to a temperature-dependent prefactor) takes the tidy dimensionless form studied in the program:

(2) f(x) = x5 / (ex − 1) This is the function whose maximum and area the program finds. It is proportional to Bλ × λ5, where Bλ is the spectral radiance per unit wavelength. See Chapter 10 for the full dimensional form.

Using the variable x achieves two things: it makes the function temperature-independent (the peak position and the normalised area do not depend on T), and it brings the domain into a convenient range — the function is negligibly small below x ≈ 0.01 and above x ≈ 50, so the range [0.01, 100] captures everything that matters.

The Exact Planck Function

For intermediate values of x — the program uses 0.05 ≤ x ≤ 20.0 as the "middle" regime — the function is evaluated exactly:

(3) f(x) = x5 / (ex − 1)    [xlow ≤ x ≤ xhigh] Implemented as planck_exact(x) in planck_physics.py. Direct evaluation is numerically safe in this range because ex − 1 is neither too close to zero nor too large to overflow.

Small-x Approximation: the Rayleigh–Jeans Limit

When x is small (long wavelengths or high temperatures, so hν ≪ kBT), the exponential can be expanded as ex ≈ 1 + x, giving ex − 1 ≈ x. The Planck function then simplifies to:

(4) f(x) ≈ x5 / x = x4    [x < xlow = 0.05] Implemented as planck_small_x(x) in planck_physics.py. This is the classical Rayleigh–Jeans regime: in dimensional form it gives Bλ ∝ T/λ4, independent of Planck's constant. It was derived from pure classical physics by Lord Rayleigh (1900) and James Jeans (1905).

At first glance the x4 growth looks alarming — if the function kept growing as λ → 0 (x → ∞) it would predict infinite total radiation, the ultraviolet catastrophe. The approximation only applies for small x; the exact formula (eq. 3) cures this by the exponential denominator.

Large-x Approximation: the Wien Exponential

When x is large (short wavelengths or low temperatures, so hν ≫ kBT), the exponential ex dominates and the −1 in the denominator is negligible. The Planck function becomes:

(5) f(x) ≈ x5 e−x = exp(5 ln x − x)    [x > xhigh = 20.0] Implemented as planck_large_x(x) in planck_physics.py. The form exp(5 ln x − x) is used rather than x5 × exp(−x) to avoid floating-point overflow when x is large. This is the Wien exponential approximation, first derived empirically by Wilhelm Wien in 1896.

The exponential decay e−x kills the Planck function rapidly for large x (short wavelengths), preventing the ultraviolet catastrophe. It is this term — arising from Planck's quantisation hypothesis — that classical physics had no way to derive.

Finding the Peak — Wien's Displacement Law

The program locates the maximum by a simple running-maximum scan: it maintains a variable y_peak (initialised to zero) and checks each new function value against it. Whenever the new value exceeds y_peak, it updates both y_peak and x_peak. In pseudocode (directly reflecting the Python in planck_driver.py): 

if y > y_peak:
    y_peak = y
    x_peak = x

When the loop is finished, x_peak contains the location of the maximum. With 20 000 steps across [0.01, 100], the step size is dx ≈ 0.005, so xpeak is accurate to about four decimal places.

The theoretical value comes from setting df/dx = 0. Differentiating equation (2) and simplifying leads to the transcendental equation:

(6) 5(1 − e−x) = x Solved numerically, the root is xpeak ≈ 4.9651. Converting back to wavelength: λmax = hc / (4.9651 kBT), giving Wien's displacement constant b = λmaxT ≈ 2.898 × 10−3 m K. Equivalently, λmax [nm] ≈ 2 898 000 / T [K].

Computing the Area — Stefan–Boltzmann

The total area under the curve is related to the total power radiated by the black body. However, the program works with f(x) = x5/(ex−1), whereas the Stefan–Boltzmann integral involves x3/(ex−1). The two are related by:

(7) x3 / (ex − 1) = [x5 / (ex − 1)] / x2 = f(x) / x2 So the area accumulated by the program is ∫ f(x)/x² dx = ∫ x³/(eˣ−1) dx. The program divides each f(x) value by x² before adding it to the running area sum — as noted in the original Schutz comment: "the area required is under the curve x³/(eˣ−1), so we must divide by x²."

The area is accumulated using the trapezoidal rule, illustrated in Figure 10.5 of the book. The program stores the previous value of the integrand in y_last, so that at each step it can add the area of the trapezoid formed by the two successive values: 

y /= (x * x)          # convert to x³/(eˣ−1)
area += 0.5 * (y + y_last) * dx
y_last = y

The exact analytic result is:

(8) ∫0∞ x3 / (ex − 1) dx = π4 / 15 ≈ 6.4939 This result involves the Riemann zeta function: ζ(4) = π⁴/90, combined with the identity ∫ x³/(eˣ−1) dx = 6ζ(4). When the program's area output is compared against π⁴/15, it provides a check on the accuracy of the numerical integration (Investigation 10.3).

With 20 000 steps the trapezoidal sum converges to this value to better than 0.01%. The Stefan–Boltzmann law M(T) = σT⁴ then follows from multiplying by the appropriate dimensional prefactor: σ = 2π⁵kB⁴ / (15 c² h³) ≈ 5.67 × 10−8 W m−2 K−4.

planck_physics.py — the three Planck functions

All three functions take a single argument x (dimensionless, as in eq. 1) and return the value of the dimensionless Planck integrand. They contain no references to physical constants or temperature — all dimensional information is handled by the caller.

FunctionFormulaRegime
planck_small_x(x) x4 x < 0.05 (Rayleigh–Jeans, long λ)
planck_exact(x) x5 / (exp(x) − 1) 0.05 ≤ x ≤ 20.0 (full Planck formula)
planck_large_x(x) exp(5 ln x − x) x > 20.0 (Wien exponential, short λ)

The form exp(5 * log(x) − x) for the large-x case is a direct transcription of the original Schutz Java: Math.exp(5.0 * Math.log(x) − x). It avoids computing x**5 and exp(−x) separately, which would risk overflow for large x.

planck_driver.py — run_planck(n_steps, output_type)

The driver function controls the sweep across the domain of x and selects the appropriate physics function at each step. The key internal variables are:

VariableValue / Role
x_min0.01 — lower bound of the sweep; f(x) is negligible here
x_max100.0 — upper bound; f(x) is effectively zero above ≈ 50
x_low0.05 — threshold below which planck_small_x is used
x_high20.0 — threshold above which planck_large_x is used
dx(x_max − x_min) / n_steps — uniform step size
x_peak, y_peakRunning maximum location and value
y_lastPrevious integrand value, needed for the trapezoidal rule
areaAccumulated trapezoidal sum of x³/(eˣ−1)

The function returns a Python dictionary: {'x_peak': ..., 'y_peak': ...} when output_type == "maximum", or {'area': ...} when output_type == "area".

main.py — the entry point

main.py is the only file the user needs to run or modify. It imports run_planck from planck_driver, calls it with the chosen parameters, and prints a clearly labelled result to the terminal. No other imports are required.

Investigations 10.1, 10.2, and 10.3

Investigation 10.1 — Qualitative Features of the Spectrum

Use the dimensional form of the Planck function (equation 2, rewritten with the prefactor 2hc²/λ⁵) to sketch the spectrum for a star at 3 000 K, 6 000 K, and 12 000 K. Without running any program: predict which curve has its peak at the longest wavelength, and which has the greatest area. Confirm your predictions using Wien's displacement law (eq. 6) and the Stefan–Boltzmann law (eq. 8). The program is not needed for this investigation.

Investigation 10.2 — The Wien Peak

Run Planck with output_type = "maximum" and the default n_steps = 20000. Record the value of xpeak. Compare it to the theoretical value 4.9651 (the root of equation 6). Then:

  1. Use Wien's displacement law λmax = hc / (4.9651 kBT) to compute the peak wavelength for the Sun (T ≈ 5 778 K) and check it falls in the visible band (400–700 nm).
  2. Repeat for the Cosmic Microwave Background (T = 2.725 K) and confirm the peak is in the microwave region (λ ≈ 1 mm).
  3. Try reducing n_steps to 200 and observe how the precision of xpeak changes. What is the minimum number of steps that still gives a result accurate to 1%?

Investigation 10.3 — The Stefan–Boltzmann Law

Run Planck with output_type = "area". The program integrates x3/(ex−1) from x_min to x_max. Compare the result to the analytical value π4/15 ≈ 6.4939. Then:

  1. Compute the Stefan–Boltzmann constant from the area: σ = (area × 2π kB4) / (c² h³ × 15/π4) and compare to the tabulated value 5.6704 × 10−8 W m−2 K−4.
  2. Use L = 4πR²σT⁴ to compute the Sun's luminosity, given R☉ = 6.96 × 108 m and T = 5 778 K. Compare to the accepted value 3.83 × 1026 W.
  3. Show that doubling a star's temperature at fixed radius increases its luminosity by a factor of 16. Verify numerically using two area calculations at T and 2T.
