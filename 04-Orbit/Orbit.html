Compute the orbit of a planet around a star in Newton’s theory of gravity. The Python version allows you to choose the mass of the central star and any desired starting position and velocity for the planet. It can output the orbit, velocity information, position or velocity as a function of time, or kinetic and potential energies. The program introduces adjustable time-step control and the predictor–corrector technique.

Description of Orbit

The Orbit program computes the motion of a planet or satellite around a central mass using Newton’s law of gravitation. It demonstrates that bound orbits are closed ellipses and allows users to explore many different physical systems by adjusting initial conditions.

The Python version faithfully follows Bernard Schutz’s original Java program from Gravity From the Ground Up, but is organized into modern, modular Python files. It supports:

  • arbitrary initial positions and velocities
  • adjustable time-step control
  • predictor–corrector iteration
  • multiple output modes (orbit, velocity space, position vs time, velocity vs time, energies)

The default initial conditions simulate Mercury’s orbit around the Sun. Users can modify these to explore other planetary systems, comets, or general gravitational motion.

Using Orbit

The Python version of Orbit is located at:
https://github.com/tomyee3/GFTGU-Programs/tree/main/Orbit

The program is divided into four modules:

  1. Physics module (physics.py):
    Encodes Newton’s law of gravitation. The gravitational acceleration is computed as: 
    ax = -k * x / r**3
ay = -k * y / r**3
    where k = G*M is the gravitational parameter.
  2. Driver module (driver.py):
    Performs numerical integration using:
    • adjustable time-step control
    • predictor–corrector iteration
    • velocity averaging for position updates
    It stores position, velocity, and energy information depending on the selected output mode.
  3. Plot module (plot_orbit.py):
    Uses Matplotlib to display:
    • the orbit in the x–y plane
    • velocity space plots
    • position vs time
    • velocity vs time
    • energy vs time
    The orbit plot enforces equal axis scaling.
  4. Main module (main.py):
    The user-facing entry point. It sets example values for:
    • initial position
    • initial velocity
    • central mass
    • initial time-step
    • maximum number of steps
    • accuracy parameters for time-step control and predictor–corrector
    • output mode
    Users may overwrite these values to explore different systems.

To run the program: 

python main.py

Depending on the selected output mode, the program will produce one or more plots showing orbital motion, velocity information, or energies.

Suggestions for playing with Orbit

The Python version allows extensive experimentation. Here are some suggestions:

  1. Planetary orbits. Use perihelion distances and maximum speeds from Table 4.3 of the book to simulate other planets.
  2. Orbital periods. Use “position vs time” or “velocity vs time” output modes to measure orbital periods and compare them with Table 4.2.
  3. Velocity periodicity. Use “velocity space” output to see that velocity repeats each orbit, confirming closed orbits.
  4. Unbound orbits. Increase initial speed to produce hyperbolic or parabolic trajectories.
  5. Energy conservation. Use “energy vs time” to verify that total energy remains constant and that bound orbits have negative total energy.

Understanding Orbit

The Python version introduces several important numerical techniques:

  1. Newton’s law of gravitation.
    The gravitational acceleration is: 
    a = -k / r**2
    with components: 
    ax = -k * x / r**3
ay = -k * y / r**3
  2. Adjustable time-step.
    The driver reduces the time-step when the fractional change in acceleration exceeds a user-specified tolerance.
  3. Predictor–corrector iteration.
    The driver iteratively refines the velocity and position updates using beginning and ending accelerations until convergence.
  4. Orbit closure detection.
    The driver uses angular information to determine when the planet has completed one orbit.

Suggested modifications

You may experiment with:

  • different accuracy tolerances
  • different predictor–corrector thresholds
  • alternative time-step control strategies
  • different gravitational parameters
