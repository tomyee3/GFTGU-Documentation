Description

CannonTrajectory is the most elementary program in the GFtGU suite. It computes the trajectory of a projectile — a cannonball, a thrown ball, or any object launched from Earth's surface — under constant gravitational acceleration, with no air resistance. It is introduced in Investigation 1.3 of Schutz's text, where it serves as the reader's first encounter with numerical integration and the concept of a program loop.

Despite its simplicity, the program introduces two genuinely distinct numerical methods: the basic Euler method and the more accurate improved Euler (Heun) method. Comparing their outputs at different timesteps gives immediate, visual intuition for the meaning of numerical accuracy and convergence — essential preparation for the more demanding integrators used in later chapters.

The original Triana Java program on which this port is based is called CannonTrajectory. The Python port preserves the same physics, loop termination logic, and array-trimming behaviour, while reorganising the code into the standard four-module structure used throughout this repository.

Physical Background

Near-surface approximation

Close to Earth's surface, the gravitational acceleration is essentially constant: \( g \approx 9.8 \;\text{m s}^{-2} \) directed downward. This approximation is excellent for trajectories whose horizontal range is small compared to Earth's radius (\(R_\oplus \approx 6.4 \times 10^6\) m). A cannonball fired at 100 m/s travels at most a few hundred metres — comfortably within this regime.

No air resistance

The program models vacuum flight. In reality, air drag would substantially reduce the range at even moderate speeds. Neglecting drag keeps the equations analytically tractable and lets the focus remain on the numerical method itself. At the muzzle velocities considered here (\(\lesssim\) a few hundred m/s), drag is important but the qualitative shape of the parabola is preserved.

Horizontal and vertical independence

Because gravity acts purely vertically and drag is absent, the horizontal and vertical motions are completely independent. Horizontally the projectile moves at constant velocity; vertically it accelerates downward at rate \(g\). This decoupling is the key insight of Galileo's parabolic-trajectory theorem, and it is preserved exactly in both the Euler and improved Euler integrators implemented here.

Governing Equations

Equations of motion

With \(x\) the horizontal distance, \(h\) the height, \(u\) the horizontal velocity, and \(v\) the vertical velocity, the continuous equations of motion are:

Eq. 1 — Horizontal kinematics
\[ \frac{dx}{dt} = u, \qquad \frac{du}{dt} = 0. \]
Eq. 2 — Vertical kinematics
\[ \frac{dh}{dt} = v, \qquad \frac{dv}{dt} = -g. \]

In the code these four rates of change are returned by derivs_cannon(state) as the NumPy array \([u,\; v,\; 0,\; {-g}]\), corresponding to the state vector \(\mathbf{s} = [x,\; h,\; u,\; v]\).

Initial conditions

The projectile is launched from the origin with speed \(v_0\) at angle \(\theta\) above the horizontal:

Eq. 3 — Launch conditions
\[ x_0 = 0, \quad h_0 = 0, \quad u_0 = v_0 \cos\theta, \quad v_0^{(v)} = v_0 \sin\theta. \]

Analytic solution

For this simple case an exact solution exists and can be used to verify the integrators:

Eq. 4 — Analytic trajectory
\[ x(t) = u_0\, t, \qquad h(t) = v_{0}^{(v)}\, t - \tfrac{1}{2} g\, t^2. \]
Eq. 5 — Range formula
\[ R = \frac{v_0^2 \sin 2\theta}{g}. \]

The range is maximised at \(\theta = 45°\), which is why the default firing angle is 45°. For \(v_0 = 100\) m/s, the analytic range is \(R = 1020.4\) m.

Euler method

The simplest time integrator advances the state by one step using only the derivative at the beginning of the interval:

Eq. 6 — Forward Euler step
\[ \mathbf{s}^{n+1} = \mathbf{s}^{n} + \Delta t\; \mathbf{f}(\mathbf{s}^{n}), \]

where \(\mathbf{f}(\mathbf{s}) = [u,\, v,\, 0,\, {-g}]\) is the derivative vector. Euler is first-order accurate: the global error is \(\mathcal{O}(\Delta t)\).

Improved Euler method (Heun's method)

The improved Euler method uses a predictor–corrector pair to achieve second-order accuracy \(\mathcal{O}(\Delta t^2)\):

Eq. 7 — Heun predictor
\[ \mathbf{s}^* = \mathbf{s}^{n} + \Delta t\; \mathbf{f}(\mathbf{s}^{n}). \]
Eq. 8 — Heun corrector
\[ \mathbf{s}^{n+1} = \mathbf{s}^{n} + \frac{\Delta t}{2}\bigl[\mathbf{f}(\mathbf{s}^{n}) + \mathbf{f}(\mathbf{s}^*)\bigr]. \]

Because the derivatives are linear in this problem (\(\mathbf{f}(\mathbf{s}^*) = \mathbf{f}(\mathbf{s}^n) + [-g\,\Delta t,\; 0,\; 0,\; 0]^T\) for the vertical component), the corrector averages the initial and final vertical velocities — identical to the update used in Schutz's original Java code. For horizontal motion, where the derivative is constant, both methods give the same result; the improvement appears only in the vertical (height) update.

Schutz's Java equivalent: In the original Triana code, the improved Euler step is written as w = v − g·dt (predictor vertical velocity), then h = h + (v + w)/2 · dt. This is algebraically identical to Eq. 8 for height, since \(\frac{v + (v - g\,\Delta t)}{2} = v - \tfrac{1}{2}g\,\Delta t\).

Algorithm

Initialisation

The launch angle is converted to radians and the velocity components \(u_0\), \(v_0\) are computed. Pre-allocated NumPy arrays xs and hs of length max_steps are filled with zeros; element 0 is set to the initial position. The integrator function (euler_step or improved_euler_step) is selected according to the method parameter.

Integration loop

The main loop advances the state vector by one timestep per iteration:

  1. Apply the chosen stepper: state = stepper(state, dt).
  2. Record xs[j] = state[0], hs[j] = state[1].
  3. Increment the counter j.
  4. Check the termination condition: state[1] < 0.0 (height went negative) or j ≥ max_steps.

Termination and array trimming

The loop exits as soon as the projectile's height becomes negative (first step that crosses the ground). The pre-allocated arrays are then trimmed to the actual number of used steps before being returned: xs[:j], hs[:j]. This avoids trailing zeros in the plot, matching the behaviour of the original Triana program.

Note on landing interpolation: The termination height is the first negative value, not zero, so the reported range is the horizontal position at the step after the projectile crosses the ground. For small \(\Delta t\) the error is negligible. Schutz's suggested Experiment 2 (below) describes how to correct this by linear interpolation to find the exact zero crossing.

Runs the simulation with default parameters and passes the result directly to the plotting function. Edit parameters here to reproduce the experiments below.

from driver_cannon import run_cannon_trajectory from plot_cannon import plot_cannon xs, hs = run_cannon_trajectory( speed = 100, angle_deg = 45, dt = 0.1, method = "improved" ) plot_cannon(xs, hs)

Parameters

Parameter Default Units Description
speed 100 m/s Muzzle speed (magnitude of initial velocity)
angle_deg 45 degrees Firing angle above the horizontal; 45° maximises range in vacuum
dt 0.1 s Fixed timestep; reduce to improve accuracy, increase to study truncation error
max_steps 1000 — Safety ceiling on loop iterations; prevents runaway for very small dt
method "improved" — Integration scheme: "euler" (1st order) or "improved" (Heun, 2nd order)
g 9.8 m/s² Surface gravitational acceleration (defined in physics_cannon.py)
Analytic check: With the defaults (speed=100, angle_deg=45), the analytic range is \(R = v_0^2 \sin 90° / g = 10000/9.8 \approx 1020\) m and the time of flight is \(T = 2 v_{0y}/g = 2 \times 70.71 / 9.8 \approx 14.4\) s, requiring \(\lceil T/\Delta t \rceil = 144\) steps — well within max_steps = 1000.

Output

The program produces a single matplotlib window titled "CannonTrajectory — Newtonian Projectile Motion" showing:

  • A blue line tracing the parabolic trajectory from launch to just below ground level.
  • An orange dot marking the launch point at the origin.
  • Equal horizontal and vertical scales (plt.axis("equal")) so the parabola's true aspect ratio is preserved — this faithfully reproduces Triana's forced-equal-ranges display mode.
  • Grid lines and a legend.

The run_cannon_trajectory function additionally returns the NumPy arrays xs and hs (each of length equal to the actual number of steps taken), suitable for further analysis or overlay plots.

Suggested Experiments

Experiment 1
Accuracy of the two integrators

Run the simulation twice with the same parameters, once with method="euler" and once with method="improved". Compare the final range (last element of xs) against the analytic value \(R = v_0^2 \sin 2\theta / g\). Then repeat with dt halved and quartered. You should find that the Euler error halves each time you halve dt (first-order), while the improved Euler error quarters (second-order). This is the numerical signature of convergence order.

Experiment 2 (Schutz suggestion)
Degrade accuracy — plain Euler for height

In improved_euler_step, replace the corrector height update with a plain Euler update: use state[1] + dt * ds1[1] instead of the averaged value. This is the "less accurate" variant Schutz describes: the height uses only the initial vertical velocity, while horizontal distance still uses the averaged value. Compare the resulting range error against the standard improved Euler result at the same timestep — and verify that the degraded version's error scales as \(\mathcal{O}(\Delta t)\) rather than \(\mathcal{O}(\Delta t^2)\).

Experiment 3 (Schutz suggestion)
Interpolated landing point

The loop terminates at the first step where \(h < 0\). Improve the accuracy of the reported range by linear interpolation: if the last two heights are \(h_{n-1} > 0\) and \(h_n < 0\), the zero crossing is at \(x^* = x_{n-1} + (x_n - x_{n-1}) \cdot h_{n-1} / (h_{n-1} - h_n)\). Add this correction to the driver and verify that it reduces the range error significantly, even for large \(\Delta t\).

Experiment 4
Optimal firing angle

Loop over firing angles from 1° to 89° in 1° steps and record the range for each. Plot range vs. angle. Confirm that the maximum occurs at 45° and that the range curve is symmetric about 45° (i.e., complementary angles give the same range). Try this with both the Euler and improved Euler methods — does the choice of integrator shift the numerically optimal angle?

Experiment 5
Effect of muzzle speed

Fix the angle at 45° and vary speed from 50 to 500 m/s. The range should scale as \(v_0^2\). Plot range vs. \(v_0^2\) on a linear scale and verify the slope equals \(\sin 90° / g = 1/g\). At very high speeds the flat-Earth, constant-\(g\) approximation breaks down — note that EarthOrbit (Chapter 4) treats this regime properly.

Experiment 6
Overlay multiple trajectories

Call run_cannon_trajectory for angles of 15°, 30°, 45°, 60°, and 75° at speed = 100 m/s and overlay all five trajectories on a single matplotlib figure. Observe that the 15°/75° pair and the 30°/60° pair each share the same range (as the analytic formula predicts). Add a horizontal line at the maximum range for reference.
