Description of CannonTrajectory

The unit called CannonTrajectory now implements a Python program for computing the trajectory of a cannonball or other projectile fired from the surface of the Earth, at an arbitrary angle and speed. The Python code closely follows Bernard Schutz’s original Java program from Gravity from the Ground Up, but is organized into separate modules for physics, numerical driver, plotting, and a user-facing main.py.

The Python source code is available at:
https://github.com/tomyee3/GFTGU-Programs/tree/main/CannonTrajectory

This program is still the most elementary one in the book, and many later programs are based on it. It is introduced in Investigation 1.3. The Python version preserves the original numerical method (simple time-stepping with a fixed time-step, no Runge–Kutta) and the pedagogical structure of Schutz’s code, while making it easier to run and modify on modern systems.

Using CannonTrajectory (Python version)

To use the Python version of CannonTrajectory, you do not need Triana or the original Java environment. Instead, you run the program directly with Python.

  1. Get the code:
    Clone or download the repository:
    https://github.com/tomyee3/GFTGU-Programs/tree/main/CannonTrajectory
  2. Install Python and dependencies:
    Make sure you have a recent version of Python (3.x) installed. The plotting module uses matplotlib, so you may need to install it:
    pip install matplotlib
  3. Set parameters in main.py:
    Open main.py in a text editor. Near the top you will find example values for:
    • dt – the time-step in seconds
    • speed – the launch speed in meters per second
    • angle_deg – the launch angle in degrees, measured from the horizontal
    These are set to example values (typically similar to Schutz’s defaults: speed ≈ 100 m/s, angle ≈ 45°, dt ≈ 0.1 s). You may overwrite these values to experiment with different trajectories.
  4. Run the program:
    From a terminal in the CannonTrajectory directory, run:
    python main.py
    The driver module will compute the trajectory and the plot module will open a window showing the path of the projectile (horizontal distance versus vertical height).
  5. Interpreting the plot:
    The plot uses equal scales on the horizontal and vertical axes by default (or can be configured to do so in the plotting code), so the trajectory appears with correct geometric proportions. You can adjust zoom and axis limits using standard matplotlib controls.

Updated documentation for the Python version of CannonTrajectory is maintained in the GFTGU-Documentation repository:
https://github.com/tomyee3/GFTGU-Documentation

Understanding CannonTrajectory (Python modules)

The Python implementation of CannonTrajectory follows Schutz’s original Java logic very closely, but separates concerns into four modules:

  1. Physics module (physics.py):
    This module encodes the physical laws governing the motion of the projectile. The only literal numbers in this module represent physical constants, such as the acceleration due to gravity:
    g = 9.8
    It provides functions that compute the acceleration and update rules for position and velocity, mirroring the simple constant-acceleration model used in the original Java program.
  2. Driver module (driver.py):
    This module performs the numerical integration of the equations of motion. It implements the same finite-difference scheme as Schutz’s Java code:
    • Horizontal motion: x = x + u * dt
    • Vertical speed update: w = v - g * dt
    • Vertical position using average speed:
      h = h + (v + w) / 2 * dt
    The driver stores the sequence of positions in arrays (or lists) and stops when either:
    • the projectile returns to the ground (h < 0), or
    • a maximum number of steps is reached (analogous to the 1000-step limit in the Java code).
    The driver returns structured output (e.g. a dictionary or a small data class) containing arrays of horizontal and vertical positions.
  3. Plot module (plot_cannon.py or similar):
    This module takes the structured output from the driver and produces a graph using matplotlib. It is responsible only for plotting: setting axis labels, titles, and visual style. The computational logic is kept separate from the plotting machinery, in contrast to the original Triana-based Java environment where plotting was integrated via the Curve data type and SGTGrapher.
  4. Main module (main.py):
    This is the user-facing entry point. It:
    • Defines example values for dt, speed, and angle_deg.
    • Calls the driver to compute the trajectory.
    • Passes the results to the plot module to display the graph.
    Users can modify the parameters in main.py without touching the physics or driver code, making experimentation straightforward.

Conceptually, the Python program still illustrates the same ideas as the original Java version: loops, numerical integration of motion under constant acceleration, and the importance of choosing an appropriate time-step. The separation into modules simply makes these ideas clearer and more reusable.

Suggested modifications of CannonTrajectory

The suggestions from Schutz’s original help file still apply, and are now easier to implement in Python:

  • Change the vertical integration method:
    In the driver module, you can replace the more accurate average-speed update:
    h = h + (v + w) / 2 * dt
    with the simpler:
    h = h + w * dt
    Then experiment with different values of dt in main.py to see how the accuracy of the trajectory depends on the time-step.
  • Improve the landing point:
    Currently, the trajectory ends when the height first becomes negative. You can refine the estimate of the horizontal distance at which the projectile actually hits the ground by interpolating between the last positive height and the first negative height. In Python, this can be done by solving a simple linear equation using the last two points and replacing the final stored point with the interpolated landing point.
  • Extend the physics:
    Because the physics is isolated in physics.py, you can add effects such as air resistance or variable gravity without changing the plotting code. This mirrors the spirit of Schutz’s suggestions for modifying the original Java program, but with a cleaner structure.

If you wish to contribute improvements or educational extensions, you can fork the GitHub repository and submit pull requests, keeping the spirit of the Creative Commons license.
