Description of Binary

The program Binary models the Newtonian orbital motion of two masses orbiting their common centre of mass. It integrates the two‑body problem using adjustable time‑step control and a velocity‑averaging predictor–corrector method, producing plots of the orbit, separation, energy, and orbital period.

This Python version matches the original Java program’s Newtonian physics exactly. It does not include gravitational‑wave energy loss or inspiral evolution. All equations and numerical techniques correspond directly to the Newtonian two‑body problem described in Chapter 13 of Gravity from the Ground Up.

Chapter Reference See Chapter 13 for binary motion, orbital energy, and period measurement. This program supports the Newtonian parts of Investigations 13.x.

Using Binary

Run the program by executing: 

python main.py

The main module defines example values for the masses, initial separation, initial velocities, time‑step parameters, and output mode. Typical parameters:

Suggestions for playing with Binary

  1. Closed Newtonian orbit. Choose initial conditions for a circular orbit. Verify closure and period.
  2. Eccentric orbits. Adjust initial velocity to produce elliptical motion. Plot separation vs time.
  3. Energy conservation. Plot total energy vs time and adjust time‑step tolerances to improve accuracy.
  4. Mass dependence. Change m1 and m2 and observe how the orbital period changes.

Understanding Binary

Keplerian Two‑Body Motion

In the centre‑of‑mass frame, the relative motion obeys:

(1) $$\frac{d^2\mathbf{r}}{dt^2} = -\,\frac{G(m_1+m_2)}{r^3}\,\mathbf{r}.$$ This is identical to the central‑force equation used in Orbit.

Energy and Angular Momentum

The Newtonian energy is:

(2) $$E = \tfrac12\,\mu\,v^2 - \frac{G m_1 m_2}{r},$$ where \(\mu = m_1 m_2 / (m_1 + m_2)\) is the reduced mass.

The driver computes \(E(t)\) at each step for plotting.

Adjustable Time‑Step and Predictor–Corrector

The driver uses the same velocity‑averaging predictor–corrector method as Orbit:

(3) Predictor: $$\mathbf{v}_{\text{pred}} = \mathbf{v}_n + \mathbf{a}_n\,\Delta t,$$ $$\mathbf{r}_{\text{pred}} = \mathbf{r}_n + \mathbf{v}_n\,\Delta t.$$ Corrector: $$\mathbf{a}_{\text{pred}} = \mathbf{a}(\mathbf{r}_{\text{pred}}),$$ $$\mathbf{v}_{n+1} = \mathbf{v}_n + \tfrac12(\mathbf{a}_n + \mathbf{a}_{\text{pred}})\Delta t,$$ $$\mathbf{r}_{n+1} = \mathbf{r}_n + \tfrac12(\mathbf{v}_n + \mathbf{v}_{n+1})\Delta t.$$ This improves accuracy without requiring Runge–Kutta.

Orbit Closure and Period Measurement

The driver monitors the angle of \(\mathbf{r}\) and detects when the binary completes a full revolution. Successive crossings give the orbital period.
